在克隆时出现“Failed to connect to github.com port 443: Timed out”

解决方案

方案1：git config --global http.sslVerify "false" 修改设置解除ssl验证

方案2：git config --global core.longpaths true 文件名或路径太长

实例

输出日志


$ git clone https://github.com/green-download.git
Cloning into 'green-download'... fatal: unable to access 'https://github.com/green-download.git/': Failed to connect to github.com port 443: Timed out
$ git config --global core.longpaths true
